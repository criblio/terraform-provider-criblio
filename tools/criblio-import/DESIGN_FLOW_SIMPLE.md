# CLI Tool Design Flow - Simple Overview

## Quick Visual Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                    CODE GENERATION (Build Time)                  │
└─────────────────────────────────────────────────────────────────┘

    openapi.yml ──┐
                  │
    overlay.yml ──┼──> [Speakeasy] ──> internal/sdk/ (API Client)
                  │                    internal/provider/ (Terraform)
                  │
    gen.yaml ─────┘


┌─────────────────────────────────────────────────────────────────┐
│                    CLI TOOL EXECUTION (Runtime)                  │
└─────────────────────────────────────────────────────────────────┘

    User Command
         │
         ├─> [Cobra CLI] ──> Parse flags/args
         │
         ├─> [Viper] ──> Load config (Flags > Env > File)
         │
         ├─> [CriblTerraformHook] ──> Auto-authenticate
         │
         ├─> [SDK Client] ──> Call API: /bulk/diag/download
         │
         ├─> [Archive Handler] ──> Extract .tar.gz
         │
         ├─> [YAML Parser] ──> Parse config files
         │
         ├─> [Resource Converters] ──> YAML → HCL
         │      ├─> Source Converter
         │      ├─> Destination Converter
         │      ├─> Pipeline Converter
         │      └─> ... (other converters)
         │
         ├─> [HCL Generator] ──> Format with hclwrite
         │
         └─> [File Writer] ──> Write .tf files
                │
                └─> [Optional: Auto-Import] ──> terraform import
```

## Detailed Step-by-Step Flow

### Phase 1: Initialization
```
User runs: criblio-import --output ./configs --include sources

1. main.go (Cobra)
   └─> Parse command and flags
   
2. initViper()
   └─> Load configuration:
       ├─> Flags: --output, --include
       ├─> Env: CRIBL_BEARER_TOKEN, CRIBL_WORKSPACE_ID
       └─> File: ~/.cribl/credentials
   
3. newImportCommand()
   └─> Create import command with all flags
```

### Phase 2: Authentication
```
4. CriblTerraformHook (automatic)
   └─> Detect auth method:
       ├─> Bearer token? → Use Bearer auth
       ├─> Client ID/Secret? → Use OAuth
       ├─> On-prem URL? → Use Basic auth
       └─> Credentials file? → Read profile
   
5. SDK Client Initialization
   └─> sdk.New() with authenticated hook
   └─> Client ready for API calls
```

### Phase 3: API Interaction
```
6. Call API Endpoint
   └─> client.System.Diag.GetDiagBundle(ctx)
   
7. Download Archive
   └─> Receive .tar.gz bytes
   
8. Extract Archive
   └─> Extract to temp directory
   └─> List YAML files
```

### Phase 4: Processing
```
9. Parse YAML Files
   └─> For each YAML file:
       ├─> Detect resource type
       ├─> Filter (if --include/--exclude)
       └─> Route to appropriate converter
   
10. Resource Conversion
    └─> Converter processes:
        ├─> Field mapping (camelCase → snake_case)
        ├─> Type conversion
        ├─> Schema validation
        └─> HCL block generation
```

### Phase 5: Output
```
11. Generate HCL Files
    └─> Create module structure:
        ├─> modules/sources/main.tf
        ├─> modules/destinations/main.tf
        └─> main.tf (provider config)
    
12. Format Files
    └─> Run terraform fmt (optional)
    
13. Optional: Auto-Import
    └─> If --auto-import:
        └─> Run terraform import for each resource
```

## Key Integration Points

### 1. SDK Integration
```
CLI Tool
  │
  ├─> import "github.com/criblio/terraform-provider-criblio/internal/sdk"
  │
  └─> client := sdk.New()
      └─> Automatic authentication via CriblTerraformHook
      └─> All API methods available
```

### 2. Provider Schema Reference
```
CLI Tool
  │
  ├─> Reference internal/provider/*_resource.go
  │
  └─> Use schemas for:
      ├─> Field names (camelCase → snake_case)
      ├─> Required vs optional fields
      ├─> Type validation
      └─> Default values
```

### 3. Converter Pattern
```
YAML Config
  │
  ├─> Parse YAML structure
  │
  ├─> Map fields (camelCase → snake_case)
  │
  ├─> Convert types (string → number, etc.)
  │
  ├─> Validate against schema
  │
  └─> Generate HCL block
```

## Error Handling Flow

```
Operation
  │
  ├─> Success? ──> Continue
  │
  └─> Error?
      │
      ├─> Auth Error ──> Show auth help
      │
      ├─> API Error ──> Show API error + troubleshooting
      │
      ├─> File Error ──> Show permission/access error
      │
      ├─> Validation Error ──> List failed resources
      │
      └─> Network Error ──> Check connectivity
```

## Summary

**Build Time:**
- OpenAPI schema → Speakeasy → Generated SDK & Provider

**Runtime:**
- User input → Config → Auth → API → Archive → YAML → HCL → Files

**Key Benefits:**
- ✅ Code reuse (SDK, auth, schemas)
- ✅ Type safety (generated types)
- ✅ Automatic updates (schema changes propagate)
- ✅ Flexible config (flags, env, files)
